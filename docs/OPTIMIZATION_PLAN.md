
# 🛠️ 项目深度优化与重构计划 (Project Optimization Plan)

## 1. 现状回顾与问题诊断 (Current Status & Diagnosis)

经过详细的代码审计和系统行为分析，目前项目处于 MVP (最小可行性产品) 阶段，虽然核心功能已跑通，但存在显著的工程隐患，导致用户体验不佳。

### 核心痛点 (Pain Points)
1.  **数据完整性差 (Data Integrity)**: 图片封面缺失、HTTPS/HTTP 混用、数值解析错误（如 "1.2万" 未能正确转换）。
2.  **系统脆弱性 (Fragility)**: RPA 强依赖 DOM 结构，缺乏降级机制；一旦小红书改版或触发反爬，系统直接报错，用户无感知。
3.  **交互反馈弱 (Weak UX)**: 后台任务（如分析、抓取）缺乏实时进度反馈，用户只能看到“结果”或“失败”，无法感知“过程中”。
4.  **架构耦合 (Coupling)**: 爬虫、数据清洗、AI 分析、数据库操作耦合在同一个 Service 函数中，难以维护和测试。

---

## 2. 深度优化路线图 (Optimization Roadmap)

### ✅ 第一阶段：数据底座治理 (Data Foundation) - [进行中]
**目标**: 确保进入数据库的数据是干净、统一、可用的。
- [x] **建立数据清洗层 (`DataSanitizer`)**: 统一处理 URL 协议、数值单位转换、文本截断。
- [x] **修复封面协议问题**: 强制所有图片链接转换为 HTTPS。
- [ ] **优化 RPA 选择器策略**: 将硬编码的选择器抽离为配置文件，便于应对页面结构变更。

### 🚀 第二阶段：异步任务架构 (Async Architecture) - [下周计划]
**目标**: 解决请求超时问题，提升系统吞吐量。
- [ ] **任务队列重构**: 确保“添加竞品”、“深度分析”等耗时操作全部走 Task Queue。
- [ ] **实时状态同步**: 实现前端轮询或 WebSocket，展示“抓取中 -> 分析中 -> 完成”的详细状态。
- [ ] **断点续传/重试**: 遇到反爬时暂停任务，提示用户介入（滑块/验证码），完成后继续任务。

### 🛡️ 第三阶段：防御性编程 (Defensive Coding) - [长期规划]
**目标**: 提升系统在异常情况下的稳定性。
- [ ] **全局错误拦截**: 统一 API 错误响应格式，前端根据错误码（ErrorCode）展示不同提示（如：`COOKIE_EXPIRED` 弹窗登录，`NETWORK_ERROR` 提示重试）。
- [ ] **降级策略**: 无法获取详细数据时，优先保留基础数据（如标题、封面），不让整个任务失败。
