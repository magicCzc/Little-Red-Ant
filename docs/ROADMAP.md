# Roadmap & Future Planning (项目规划)

本文档概述了 **Little Red Ant (小红蚁)** 项目的后续开发计划、功能扩展方向以及需要优化的技术债务。

## 📍 Phase 1: 稳定性与工程化 (当前阶段)
> 目标：巩固现有功能，确保“生成-发布-统计-互动”核心链路 100% 可靠，无阻塞。

- [x] **架构重构**：完成 AI Provider 模式，支持多模型切换。
- [x] **异步任务系统**：所有耗时操作（发布、生成、爬虫）迁移至 Task Queue + Worker。
- [x] **RPA 内核升级 (v2.0)**：
    - [x] **BrowserService**：统一浏览器会话管理，实现 Cookie 自动注入与多账号隔离。
    - [x] **智能交互**：实现“混合滚动”(Hybrid Scroll)与“辐射定位”(Radial Search)，彻底解决懒加载与动态 DOM 问题。
    - [x] **抗干扰**：拟人化输入 (Human-like Typing) 与 状态感知 (State Awareness)，有效规避风控与操作冲突。
- [x] **编译修复**：消除 TypeScript 类型错误，统一前端组件库 (Lucide React)。
- [ ] **错误处理增强**：
    - [x] **全局异常捕获 (Global Error Boundary)**：防止单点故障导致整个 Node 进程崩溃。
    - [x] **任务失败的自动重试策略 (Backoff Retry)**：指数退避重试 (1m, 2m, 4m)，最大重试 3 次。
- [x] **日志系统**：已实现 `LoggerService`，支持文件与控制台双重输出，关键 RPA 操作自动截图留证。

## 📍 Phase 2: 功能深度拓展 (Next Step)
> 目标：丰富内容形式，提升账号管理的精细度。

### 1. 💬 深度互动中心 (Engagement Center Pro)
- [x] **全量同步**：支持“无限滚动”抓取，自动回溯 7 天内的所有评论与交互。
- [x] **精准回复**：解决“幽灵回复”问题，支持对任意历史评论进行定位回复。
- [x] **意图识别 (Intent AI)**：
    - 自动分析评论情感 (正向/负向/咨询)。
    - 识别“高价值线索”（如：求链接、怎么买），自动标记并通知。
- [ ] **私信接管**：从公开评论延伸到私信自动回复 (需更高的风控对抗等级)。

### 2. 🛡️ 账号健康与安全 (Account Health)
- **需求**：防止账号因 Cookie 过期或频繁操作被封禁。
- **方案**：
    - [x] **Cookie 智能保活**：BrowserService 自动检测 Cookie 有效性，并支持主站/创作中心 Cookie 互备。
    - [x] **指纹混淆**：引入 `playwright-stealth` 或 Canvas 指纹噪音，防止被识别为机器人。
    - [x] **拟人化操作**：鼠标贝塞尔曲线移动、随机犹豫时间、模拟真实点击反馈。
    - [x] **操作频率限制**：在 Worker 中实现 Daily Limit，限制单账号每日发布数量（默认5篇/天）。

### 3. 📅 内容日历 (Content Calendar)
- **需求**：用户希望规划一周的内容，而不是生成一条发一条。
- **方案**：
    - [x] **待发布队列**：支持 Scheduled Tasks，按时自动发布。
    - [x] **日历视图**：可视化展示发布排期与任务状态。

### 4. � 矩阵化运营 (Matrix Operations)
- **需求**：支持同时管理多个账号、多套人设，实现流水线式内容生产。
- **方案**：
    - [x] **多账号管理**：独立管理每个账号的 Cookie、状态和权限（创作中心/主站）。
    - [x] **多相关人设**：解耦账号与人设，支持创建无限个“内容马甲”（如：美妆号、宠物号）。
    - [x] **去 AI 化引擎**：重构生成 Prompt，彻底屏蔽 AI 常用词，支持“闺蜜唠嗑”、“疯狂安利”等特调风格。
    - [x] **竞品监控 Pro**：自动拆解对标账号的爆款逻辑与关键词。

### 5. �💬 评论区互动 (Auto Reply)
- [x] **需求**：提高账号活跃度。
- [x] **方案**：
    - [x] 爬取新笔记的评论。
    - [x] 调用 AI 根据评论内容生成回复建议。
    - [x] 人工审核后一键回复。

## 📍 Phase 3: 商业化与协作 (Long Term)
> 目标：支持多用户、多团队协作，具备 SaaS 化雏形。

- [x] **多用户系统**：
    - [x] **认证中心**：集成 `bcrypt` + `JWT`，实现注册/登录/Token 校验。
    - [x] **权限控制 (RBAC)**：实现 `admin` 与 `editor` 角色分离，后台增加用户管理面板。
    - [x] **前端适配**：新增登录页，自动拦截未授权访问并跳转；退出登录增加二次确认弹窗。
- [ ] **竞品监控**：深度分析对标账号的爆款逻辑（关键词、封面风格、选题方向）。
    - [x] **深度分析**：AI 自动拆解对标账号的爆款逻辑（关键词、封面风格、选题方向）。
    - [x] **策略生成**：根据竞品数据生成 3 条可执行的模仿建议。
- [ ] **小程序/移动端**：开发配套的小程序，方便在手机上查看数据和审核发布。

## 🔧 技术债务 (Tech Debt)
- **测试覆盖**：目前缺乏单元测试，建议引入 `Vitest` 对核心 Service (AI, Queue) 进行测试。
- [x] **Docker 化**：提供 Dockerfile 和 docker-compose.yml，解决 Playwright 依赖环境不一致的问题。
- [x] **CI/CD**：配置 GitHub Actions 进行自动构建和类型检查。
